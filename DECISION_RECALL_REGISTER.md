# Decision Recall Register — Pepperyn L6

**Ouvert le :** 2026-07-20
**Principe :** Revenir sur chaque décision recommandée par Pepperyn 6 mois après l'analyse.
**Objectif :** Mesurer si la décision était la bonne — pas si le rapport était beau.
**Usage :** Ce registre alimente le Decision Capital. Sans lui, le capital reste théorique.

> La seule façon de savoir si Pepperyn améliore les décisions est de regarder
> ce qui s'est passé réellement. L'observation différée est irremplaçable.

---

## Questions de recall (5 par décision)

| # | Question |
|---|---------|
| R1 | La décision a-t-elle été prise ? (Oui / Non / Partiellement) |
| R2 | Si oui, a-t-elle produit l'effet attendu ? (Oui / Non / Partiellement) |
| R3 | L'impact financier estimé était-il dans le bon ordre de grandeur ? (±25% / ±50% / hors échelle) |
| R4 | Regrettez-vous d'avoir suivi cette recommandation ? (Oui / Non) |
| R5 | Cette analyse a-t-elle changé une décision que vous auriez autrement prise ? (Oui / Non) |

## Synthèse Decision Capital (à compléter lors du recall)

```
Décisions effectivement prises     : /10
Décisions ayant produit l'effet    : /N_prises
Delta impact moyen                 : ±?%
Décisions regrettées               : /10
Analyses ayant changé une décision : /10

Verdict Decision Capital :
  Pepperyn améliore les décisions   : OUI / NON / MITIGÉ
  Fiabilité estimations financières : BONNE / ACCEPTABLE / FAIBLE
```
